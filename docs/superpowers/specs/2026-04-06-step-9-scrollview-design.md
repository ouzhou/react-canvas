# 阶段四 Step 9 — ScrollView 与区域滚动（V1）— 设计规格

**日期：** 2026-04-06  
**状态：** 草案；与 [development-roadmap.md](../../development-roadmap.md) 阶段四 Step 9 及 brainstorming 结论一致。  
**范围：** 画布内 **`<ScrollView>`**：**垂直滚动**为 V1 必达；**拖拽与滚轮**更新内容偏移；**绘制**与 **命中测试**与 **`overflow` 裁剪**一致。**不含** 惯性滚动、嵌套滚动容器、弹性与滚动条（与路线图表格中「可选」项对齐为后续迭代）。**水平滚动**：API 可预留 `horizontal`，V1 **可不实现或 no-op**，须在用户文档标明。

---

## 1. 目标与非目标

### 1.1 目标

- 提供与路线图验收示例一致的用法：在固定**视口高度**内展示可纵向滚动的子内容（如多行 `Text`）。
- **Core**：滚动容器在场景树中有明确语义；**`contentOffset`（或等价 `scrollX` / `scrollY`）** 参与 **绘制**（视口裁剪后，对内容子树施加与偏移一致的变换）与 **命中测试**（同偏移的逆映射 + 与可视区域一致的裁剪）。
- **React**：`<ScrollView>` 宿主与 reconciler 集成；在 **canvas 指针与 `wheel`** 路径上更新偏移，并优先走 **重绘队列**（`queueLayoutPaintFrame` 或经评审后等价的「仅 paint」路径），避免仅为滚动而触发不必要的 React reconcile。
- **布局**：由 Yoga 量测得到 **视口尺寸** 与 **内容尺寸**，导出 **滚动边界**（如 `maxScrollY = max(0, contentHeight - viewportHeight)`），并在布局变化时 **钳制**当前偏移。

### 1.2 非目标（V1 不承诺）

- **惯性滚动**、**弹性回弹**、**滚动指示器**：路线图表格中列为任务或可选项，**不纳入 V1 验收**。
- **嵌套滚动**（外层 `ScrollView` 内再套 `ScrollView`）：**不支持**；规格写明，不验收。
- **`Animated.event` / 与 `contentOffset` 的动画联动**：与 [animation-system-design.md](./2026-04-06-animation-system-design.md) 一致，**不依赖** V1 ScrollView。
- **`onScroll`**：**非 V1 必达**；若实现则仅为回调，不引入动画系统耦合。

## 3. 推荐架构（方案 1）

### 3.1 与其它方案对比（简）

| 方案                                                    | 说明                                                                    | 结论                             |
| ------------------------------------------------------- | ----------------------------------------------------------------------- | -------------------------------- |
| **1. 独立 `ScrollViewNode`（或 `type: "ScrollView"`）** | 偏移存于场景节点；`paint` / `hit-test` 分支与 `View` 共享 Yoga/样式基础 | **采用**                         |
| 2. 通用「可滚动 View」                                  | 能力堆在 `ViewNode` 上                                                  | 职责膨胀，易与普通过 `View` 混淆 |
| 3. 仅双 `View` + transform                              | 偏移不进统一场景语义                                                    | 命中/滚轮与布局边界易不一致      |

### 3.2 分层职责

| 层级                    | 职责                                                                                                                                                                                                                                                                                                                                                                                              |
| ----------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **@react-canvas/core**  | 滚动容器节点类型；**绘制**：在现有 **`overflow: hidden`** 与圆角等裁剪语义之后（与 [paint.ts](../../../packages/core/src/render/paint.ts) 一致），对**内容子树**施加 **translate(-scrollX, -scrollY)**（或等价矩阵）；**命中**：[hit-test.ts](../../../packages/core/src/input/hit-test.ts) 扩展为对滚动容器内点做 **内容坐标**映射并遵守相同可视边界；**滚动边界**由布局结果导出；**钳制**偏移。 |
| **@react-canvas/react** | [hosts/scroll-view.tsx](../../../packages/react/src/reconciler/host-config.ts) 落点（见 [react-package-structure-design.md](./2026-04-07-react-package-structure-design.md)）；host config 注册；指针拖拽与滚轮与现有 **canvas 指针**管线集成。                                                                                                                                                   |

### 3.3 数据流（概念）

1. 用户 **拖拽** / **滚轮** → 更新节点上 **`contentOffset`**（V1 以 **纵向分量**为主）。
2. 调用 **重绘请求**（与现有 layout/paint 帧一致）。
3. **布局变更**（子树高度变化）→ 重算 **maxScroll** → **钳制** `contentOffset`。

---

## 4. 布局与树结构

- **视口**：`ScrollView` 节点自身布局框为**可见区域**（由 `style` 如 `height` 等决定）。
- **内容**：V1 约定用户结构可与路线图一致：**单个子树**作为滚动内容（例如包裹多行 `Text` 的 `View`）；该子树的 **量测高度** 决定 **content 尺寸**。具体 Yoga 节点挂载方式（单子节点 vs 片段）在实现计划中细化，但必须满足：**内容高度可大于视口高度** 时 **`maxScrollY > 0`**。
- **单层滚动**：子树内 **不得再**包含另一 `ScrollView` 作为 V1 支持场景；若用户书写，行为 **未定义**、不验收。

---

## 5. 输入语义（V1）

- **指针拖拽**：在滚动容器视口内按下并移动时，按位移增量更新偏移（轴与 `horizontal` 预留策略一致；V1 实现纵向）。
- **滚轮**：在指针位于滚动容器上方时，`wheel` **delta** 映射为偏移变化；**preventDefault** 等行为与阶段三「wheel 未定」的衔接在实现时与 [phase-3-interaction-design.md](./2026-04-05-phase-3-interaction-design.md) 交叉评审，避免整页滚动与画布滚动冲突。

---

## 6. 验收标准（V1）

- 与 roadmap 示例等价的 **纵向列表**可滚动：**拖拽**与 **滚轮**均能改变可见区域。
- **点击 / 命中**：滚动后，`onClick` / 命中目标与**视觉上**的子节点一致（与偏移、裁剪一致）。
- **边界**：滚到头/底时偏移被 **钳制**，不出现内容无限移出或命中错位。
- **文档**：若暴露 `horizontal`，标明 V1 **未实现或 no-op**。

---

## 7. 测试策略

- **Core**：固定小场景树，覆盖 **paint 与 hit-test** 在 **非零 contentOffset** 下的一致性与圆角/ `overflow` 组合（与现有 [hit-test 测试](../../../packages/core/tests/input/hit-test.test.ts) 风格一致）。
- **React**：至少 **一个**端到端或集成冒烟：挂载含 `ScrollView` 的片段，断言滚动后交互仍正确（若仓库已有同类测试模式则对齐）。
