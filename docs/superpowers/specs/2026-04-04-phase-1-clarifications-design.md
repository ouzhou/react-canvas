## 2. 已确认决策摘要

| #   | 主题                      | 决议                                                                                                                                                            |
| --- | ------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| 1   | **`style` 形态**          | 阶段一 **仅支持对象**；不支持 `style` 数组；类型与文档需写明。                                                                                                  |
| 2   | **Reconciler 根**         | **隐式容器根（A）**：用户写法保持 `<Canvas><View>…</View></Canvas>`；根上 `calculateLayout` 使用 `Canvas` 的 `width`/`height`（逻辑像素）。                     |
| 3   | **`<canvas>` 与 Surface** | **优先 `ref` 取 DOM** 交给 CanvasKit；若所用 `canvaskit-wasm` API 仅支持字符串，则用 **唯一 `id` 兜底**。                                                       |
| 4   | **百分比宽高**            | **支持**常见 `width`/`height` 等百分比字符串；**相对父布局盒**；**第一层宿主 `View` 的百分比相对 `Canvas` 逻辑宽高**。                                          |
| 5   | **`Canvas` children**     | **仅允许单个子节点**（阶段一为单个 `<View>`）；多子须外包一层 `View`；多顶层 / Fragment 多宿主并列 → **运行时失败**（对齐运行时结构约束文档中的严格校验策略）。 |

## 3. 对测试与验收的影响

- **导入约定**：新增测试从 **`vite-plus/test`** 导入 API，由 **`vp test`** 执行（与 `AGENTS.md` 一致）。
- **百分比**：`packages/core/tests` 增加至少一组「父定宽、子 `50%`」类断言。
- **单子约束**：`packages/react/tests` 增加非法多子（或 Fragment 双 `View`）**期望抛错** 的用例。
- **`style` 数组**：阶段一不测数组；若误传，以实现约定为准（忽略或抛错需在实现里统一，建议 **dev 抛错**）。
