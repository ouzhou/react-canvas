## 摘要

`ViewStyle` 支持 **`zIndex?: number`**，用于在同一**父节点**的多个子节点之间控制**谁画在上层**、**命中谁优先**。未设置时视为 **`0`**。

实现上 **`zIndex` 不参与 Yoga 布局**：`node.children` 与 Yoga 子树顺序保持与 React 一致；仅在 **`paintNode`** 与 **`hitTest`** 遍历子节点时，按 `zIndex` 与声明顺序做一次**稳定排序**。

---

## 动机

在仅按 **DOM/React 子顺序**绘制时，**后声明的兄弟节点**永远盖在先声明的上面。浮层组件（如 **`Select`** 下拉）若排在兄弟前面，容易被后面的 `View` 遮住。

引入 `zIndex` 后，可在**不调整组件树插入顺序**的前提下，让整块子树（例如展开中的下拉）叠在兄弟之上。

---

## 语义

| 项目         | 说明                                                                                                                                |
| ------------ | ----------------------------------------------------------------------------------------------------------------------------------- |
| **作用范围** | 仅 **同一父节点** 的**直接子节点**之间比较；不是浏览器里跨层级的完整 CSS stacking context 模型。                                    |
| **排序键**   | 先按 **`zIndex` 升序**（数值小的先绘制，数值大的后绘制 → 视觉上在上层）；`zIndex` 相同则保持 **React 子节点顺序**（与 Yoga 一致）。 |
| **命中顺序** | 与绘制一致：自顶向下递归时，对子节点按 **与绘制相同的排序**，再 **从后往前**探测（后绘制的先被命中）。                              |
| **默认值**   | 未设置或非有限数字 → 按 **`0`** 参与比较。                                                                                          |

---

## 实现要点（`@react-canvas/core`）

1. **样式拆分**  
   `zIndex` 放在 **`VISUAL_KEYS`** 中，经 **`splitStyle`** 进入 **`ViewNode.props`**，**不**写入 Yoga 的 flex/尺寸等布局键。

2. **排序函数**  
   **`getSortedChildrenForPaint(node)`**（`render/children-z-order.ts`）对 `node.children` 做拷贝排序，**不修改**原数组，因此 **Yoga 的 `insertChild` 顺序不变**。

3. **绘制**  
   **`paintNode`** 在递归普通 `View`、**`Image`**、**`SvgPath`** 子树时，子节点迭代使用 **`getSortedChildrenForPaint(node)`** 的结果顺序。

4. **命中**  
   **`hitTestRecursive`** 对同一父节点的子节点使用**同一套排序**，再逆序遍历，保证与「后绘在上」一致。

5. **对外导出**  
   **`getSortedChildrenForPaint`** 可从 **`@react-canvas/core`** 导入，便于测试或调试；一般业务只需在 **`style`** 上设 `zIndex`。

---

## 示例

```tsx
<View style={{ flexDirection: "column" }}>
  <View style={{ height: 40, backgroundColor: "#eee" }} />
  {/* zIndex 更高，即使写在上面也会盖住下面的重叠区域 */}
  <View
    style={{
      position: "absolute",
      top: 20,
      left: 0,
      right: 0,
      height: 80,
      zIndex: 10,
      backgroundColor: "#4488ff",
    }}
  />
  <View style={{ height: 100, backgroundColor: "#ccc" }} />
</View>
```

## 与 UI 组件

**`@react-canvas/ui` 的 `Select`** 在展开时会给根容器设置 **`zIndex`**，并在**每次展开**时用 **`useAllocateOverlayZIndex()`**（来自 **`@react-canvas/react`**，按 **`CanvasProvider` 实例**计数；无 Provider 时回退 **`allocateOverlayZIndex()`**）取递增整数，使**最后打开**的下拉在同父兄弟之间叠在最上。若仍被更外层节点遮挡，需在更上层布局中调整兄弟的 `zIndex` 或结构。

固定 **`zIndex: 1000`** 不足以处理「多个浮层同时打开且 `zIndex` 相同」的情况：此时仍按子节点声明顺序绘制，**后声明的实例会盖住先声明的**（见 [children-z-order 实现](https://github.com/ouzhou/react-canvas/blob/main/packages/core/src/render/children-z-order.ts) 中「同 `zIndex` 保持 React 顺序」）。
