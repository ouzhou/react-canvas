`@react-canvas/ui` 通过 **`CanvasThemeConfig`** 描述主题：种子色 **`seed`**、外观 **`appearance`**、密度 **`density`**，以及可选的 **`components`** 级覆盖。解析结果为 **`CanvasToken`**（间距、字号、颜色等），供 `Button` 等组件与 `mergeViewStyles` / `resolveSx` 使用。

## `CanvasThemeConfig`

| 字段         | 类型                       | 说明                                                                       |
| ------------ | -------------------------- | -------------------------------------------------------------------------- |
| `seed`       | `Partial<SeedToken>`       | 可选。`colorPrimary`、`borderRadius` 等种子值；未填项使用库内默认种子。    |
| `appearance` | `"light"` \| `"dark"`      | 可选，默认 `"light"`。暗色在 **已含紧凑后的 token** 上叠加（见下文顺序）。 |
| `density`    | `"default"` \| `"compact"` | 可选，默认 `"default"`。**紧凑**会缩小部分 `padding*` 与 `fontSize*`。     |
| `components` | `ComponentTokenMap`        | 可选。按组件维度再合并进 token（如 `Button`）。                            |

在代码中取得扁平 token 可使用 **`getCanvasToken(config)`**；在 React 树中可使用 **`CanvasThemeProvider`** + **`useCanvasToken()`**。

## Token 生成顺序

`getCanvasToken` 的合并顺序与实现一致（见 `packages/ui/src/theme/get-canvas-token.ts`）：

1. **`defaultAlgorithm(seed)`**：由种子生成基础 `CanvasToken`（亮色背景、正文色、默认内外边距与字号等）。
2. **`compactAlgorithm`**：当 **`density === "compact"`** 时，在**当前 token** 上叠紧凑覆盖（见下一节）。
3. **`darkAlgorithm`**：当 **`appearance === "dark"`** 时，在**当前 token**（已含 compact）上叠暗色背景、边框与主色提亮等。
4. **`components`**：最后合并组件级 token。

因此 **紧凑先于暗色**：`{ appearance: "dark", density: "compact" }` 与仅暗色相比，内边距与字号仍遵循紧凑规则。

## `CanvasThemeProvider` 与 `useCanvasToken`

- **`CanvasThemeProvider`** 接收 **`theme={CanvasThemeConfig}`**，与父级主题 **`mergeThemeConfig`** 后生成上下文中的 **`CanvasToken`**。
- **`useCanvasToken()`** 必须在 Provider 内使用，返回当前合并后的 token。

## 紧凑模式（`density: "compact"`）

**`compactAlgorithm`** 在默认 token 上调整（实现见 `compactAlgorithm`）：

- **`paddingSM` / `paddingMD`**：约为原值的 **0.75**，并设下限（避免过小）。
- **`fontSizeSM` / `fontSizeMD`**：在原有基础上 **减 1**，并设下限。

用于信息密度较高的界面；与 **`appearance`** 正交，可组合使用。

## 暗色（`appearance: "dark"`）

**`darkAlgorithm`** 调整背景、文字、边框色，并对 **`colorPrimary`** 做提亮，避免主按钮与深色背景融在一起。

## 画布内使用主题

`<Canvas>` 使用 **独立的 react-reconciler 根**，**不会**继承外层 `CanvasThemeProvider` 的 React Context。在画布内使用 `Button` 等依赖 token 的组件时，须在 **DOM 侧**用 **`useCanvasToken()`**（或 **`getCanvasToken(config)`**）取得 **`token`**，再 **`token={token}`** 传入。

<Aside type="tip">
  与 **`CanvasProvider`** 的关系：画布仍须包在 **`CanvasProvider`** 内以加载 Yoga 与
  CanvasKit；主题由 **`CanvasThemeProvider`** 提供，二者可同时使用，顺序不限，但画布内需显式传
  **`token`**。
</Aside>
